Redux About

Redux Install

- npm install react-redux

Actions

- Actions send data from your application to your store

- Actions are plain JavaScript objects

- Redux, action creators simply return an action:

- Actions are central to Redux

- An action has two properties, type and payload

- Only type is required

- you can put as many properties on an object

Reducers

- Reducers specify how the application's state changes in response to actions sent to the store.

- The reducer is a pure function that takes the previous state and an action, and returns the next state.

Store

- The Store is the object that brings them together. The store has the following responsibilities:

- Holds application state;

- Allows access to state via getState();

- Allows state to be updated via dispatch(action);

- Registers listeners via subscribe(listener);

- It's important to note that you'll only have a single store in a Redux application.

Connect

- The connect() function connects a React component to a Redux store.

- It provides its connected component with the pieces of the data it needs from the store, and the functions it can use to dispatch actions to the store.

Router

There are two types of router object.

- 1) BrowserRouter and

- If we want to handle the dynamic request then use BrowserRouter

- 2) HashRouter

- if we want to serve the static request then use HashRouter.

NavLink

- NavLink helps us conditionally add styling attributes to the rendered element if it matches the current URL.

Route

- A route can be placed anywhere inside the Router.

ActiveClassName

- By default, the class name ‘active’ is applied to the active NavLink without any need to specify the activeClassName prop.

UseLocation

- The useLocation hook returns the location object that represents the current URL.
